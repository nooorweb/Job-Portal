2. 📚 Syllabus (Structured Learning Path — NOT a blog)
This is not an article. It's a week-by-week curriculum tied to a specific job. Think of it like a university course outline or a Udemy course curriculum — structured, visual, interactive.
/jobs/[slug]/syllabus  → The learning path for that job role
Each Syllabus page has:

Weeks/Modules (accordion or timeline)
Topics per week
Resource links (external docs, videos)
Progress tracker (localStorage)
"Ready to take the quiz?" CTA at the bottom


3. 📰 Content Pages — "Article-Type but NOT WordPress"
This is the most important clarification. Here's exactly what you mean:
❌ WordPress Style✅ Your StyleBlog posts with categories/tagsStandalone in-depth guidesAuthor bio, comments sectionClean reading experience, no clutterSidebar widgets, adsFull-width, focused reading/blog/post-title pattern/learn/topic-name or /guide/topic-nameCMS-managed HTML blobsMDX files (Markdown + React components)Generic post templateEach guide has its own layout feel
What it actually is: Think Stripe Docs, Vercel Guides, Josh Comeau's blog, or The Pragmatic Engineer — long-form, beautifully typeset, standalone knowledge pages. Written in MDX (.mdx files), rendered as Next.js pages.
/learn/[slug]          → A deep-dive guide (e.g., "What is React?")
/guides/[slug]         → A career guide (e.g., "How to crack a frontend interview")
Each Content Page has:

Clean reading layout (max 680px width, centered)
Table of contents (sticky sidebar, auto-generated from headings)
Estimated read time
Related jobs CTA at the bottom
Custom MDX components — callout boxes, code blocks, comparison tables, tip/warning boxes — not WordPress shortcodes
